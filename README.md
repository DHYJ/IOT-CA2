What is SmartPark about?</h3>
<p>SmartPark is a showcase of what a Smart Carpark can look like. It has a few features such as
recommending an empty parking lot to a user that is the nearest to a lobby (for malls). It also has led
lights to indicate if the lot is taken or not, which allows users to see from afar if that section of the
carpark is taken so that he/she does not have to drive there and try to find the lots. Users can also
visit the web interface that is accessible by both computer or mobile to check which lot they park
their car in, in the event that they have forgotten. Apart from that, parking history can also be viewed
online to see which carpark did they access previously and details such as time and pricing will also
be there. Other features which normal carparks also have are free first 10 minutes parking, which
will be scaled down to 15 seconds for demonstration purposes.</p>

<p>a) Light-Dependant Resistor (LDR) are light sensitive resistors which
change resistance based on how much light they are exposed to. In
this case, the resistance is higher when it is dark. We will use this
hardware to tell whether or not a vehicle is parked. This will be
done by placing the resistor below the vehicle. Based on testing,
the raw values from the MCP3008 ADC will be &gt;0.62 if an object is
above it.</p>

</code></pre>
<p>b) Edit the .profile to include the path of the AWS client</p>
<pre><code>sudo nano ~/.profile
</code></pre>
<p>c) Add in the following code after the last line and save the file</p>
<pre><code>export PATH=~/.local/bin:$PATH
</code></pre>
<p>d) Type the following command at the command-line prompt to make the new settings take effect immediately</p>
<pre><code>source ~/.profile
</code></pre>
<p>e) Create a file named iot-role-trust.json in the same folder as the four certifcate and key

<pre><code>{^
"Version":"2012- 10 - 17",
"Statement":[{
"Effect": "Allow",
"Principal": {
"Service": "iot.amazonaws.com"
},
"Action": "sts:AssumeRole"
}]
}
</code></pre>
<p>f) Create a file named iot-policy.json in the same folder as the four certifcate and key files
with the following contents:</p>
<pre><code>{^
"Version": "2012- 10 - 17",
"Statement": [
{
"Effect": "Allow",
"NotAction": [
"iam:*",
"organizations:*"
],
"Resource": "*"
},
{
"Effect": "Allow",
"Action": [
"iam:CreateServiceLinkedRole",
"iam:DeleteServiceLinkedRole",
"iam:ListRoles",
"organizations:DescribeOrganization"
],
"Resource": "*"
}
]
}
</code></pre>

<pre><code>sudo pip install awscli
</code></pre>

<h5><a id="user-content--enable-spi-via-raspi-config-" class="anchor" aria-hidden="true" href="#-enable-spi-via-raspi-config-"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>&lt;&lt; Enable SPI via raspi-config &gt;&gt;</h5>

<pre><code>sudo rasp-config
</code></pre>

<h5><a id="user-content--enable-device-tree-in-boottxt" class="anchor" aria-hidden="true" href="#-enable-device-tree-in-boottxt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>&lt;&lt; Enable device tree in boot.txt&gt;&gt;</h5>
<p>a) Modify the /boot/config.txt to enable SPI</p>
<pre><code>sudo nano /boot/config.txt
</code></pre>
<p>b) Ensure these lines are included in config.txt</p>
<pre><code>device_tree_param=spi=on
dtoverlay=spi-bcm2835
</code></pre>
<p><strong>&lt;&lt; Install Python-dev&gt;&gt;</strong>
c) Install the Python development libraries</p>
<pre><code>sudo apt-get install python-dev
</code></pre>
<p><strong>&lt;&lt; Install SPI-Py Library &gt;&gt;</strong></p>
<p>d) Set up the SPI Python libraries since the card reader uses the SPI interface</p>
<pre><code>git clone https://github.com/lthiery/SPI-Py.git
cd /home/pi/SPI-Py
sudo python setup.py install
</code></pre>
<p><strong>&lt;&lt; Install RFID library &gt;&gt;</strong></p>
<p>e) Clone the MFRC522-python library and copy out the required files to your project directory</p>
<pre><code>git clone https://github.com/rasplay/MFRC522-python.git
cd MFRC52 2 - python
sudo cp *.py ~/SmartPark
</code></pre>
<p>f) Edit the MFRC522.py file that you just cloned from GitHub.</p>
<pre><code>sudo nano ~/SmartPark/MFRC522.py
</code></pre>

<h2><a id="user-content-section-8-program-setup" class="anchor" aria-hidden="true" href="#section-8-program-setup"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Section 8 Program setup</h2>

<h3><a id="user-content-a-installing-necessary-libraries" class="anchor" aria-hidden="true" href="#a-installing-necessary-libraries"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>A. <h3><a id="user-content-b-entrypy" class="anchor" aria-hidden="true" href="#b-entrypy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>B. entry.py</h3>
<p>This python file uses both the LCD and RFID Sensors to act as an entry gate. It will scan for an rfid
card and will insert a database entry to the dyanmodb table <strong>only when the car is not active in the
carpark</strong>. If the car is not active, it will insert an entry and the status of the car would be ‚Äòactive‚Äô as it
is in the carpark. It will also retrieve the recommended lot that is determined based on the distance
the lot is from the lobby <strong>only if the lot is empty</strong>.</p>
<p>Do note that the Carpark name should be changed based on your database. This can be changed by
modifying the string <strong>‚Äòcarparkname‚Äô</strong>.</p>
<h3><a id="user-content-c-carparkname_lot_lotnumberpy" class="anchor" aria-hidden="true" href="#c-carparkname_lot_lotnumberpy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>C. CARPARKNAME_Lot_LOTNUMBER.py</h3>
<p>Parking lot should be able to detect if the car is there if the RFID card is detected and the light level
is low. Once this occurs, the light should turn red and the Parking lot table will have a current
holder (value of UID), the available lots for the carpark should also decrease. Once the car leaves,
the reverse will happen where the available lots will go back up and the parking lot will appear as
None after the 20 seconds timeout.</p>
<p>Do note that the Carpark name should be changed based on your database. This can be changed by
modifying the string <strong>‚ÄòCarpark‚Äô</strong>. As parking lots cannot have the same name, we would have to
change it. This can be changed by modifying the string <strong>‚ÄòLot_Scanner‚Äô</strong>. For this example, we showed
the example of <strong>Bukit Batok Carpark</strong> with <strong>Lot A1</strong>. Hence the naming convention will be
<strong>BB_Lot_A1.py</strong></p>
<h3><a id="user-content-d-exitpy" class="anchor" aria-hidden="true" href="#d-exitpy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>D. exit.py</h3>
<p>This python file uses both the LCD and RFID Sensors to act as an exit gate. It will scan for an rfid
card and determine the price. When a card is detected, It will display the price and change the
database entry of the car‚Äôs status to be inactive. The price that the user paid will also be inserted
into the entry so that the user can see his/her parking history. In normal carparks the first 10
minutes has a free entry but for demonstration purpose we made it 15 seconds.</p>
<p>Do note that the Carpark name should be changed based on your database. This can be changed by
modifying the string <strong>‚Äòcarparkname‚Äô</strong>.</p>
<h3><a id="user-content-e-getdatapy" class="anchor" aria-hidden="true" href="#e-getdatapy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>E. getData.py</h3>
<p>This python file will get and send the lot ID to the web interface based on current holder ID
received. The program will subscribe to topic sensors/currentHolder and wait for incoming current
holder ID. Once it receives a current holder ID from topic sensors/currentHolder, it will access the
DynamoDB table called Parking_Lots and retrieve the lot ID based the current holder ID received.
Afterwards, it will publish to the topic sensors/lotid to be received by server.py.</p>
<h3><a id="user-content-f-serverpy" class="anchor" aria-hidden="true" href="#f-serverpy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>F. server.py</h3>
<p>This python file will host the web interface via flask program. The web interface can retrieve the lot
ID based on the current holder ID sent and retrieve the parking history based on the current holder
ID and the carpark name sent. For the lot ID, the program will publish to topic
sensors/currentHolder based on the current holder ID sent from the web interface. The program
will at the same time subscribe to sensors/lotid and wait for incoming lot ID. Once it receives the lot
ID, it will be displayed in the web interface. For parking history, it will access the DynamoDB table
called Parking and retrieve all the current status, timestamp, carpark name and price paid based on
the current holder ID and the carpark sent from the web interface. These retrieved data will be
displayed in the web interface in table format.</p>

<h2><a id="user-content-section-10-expected-outcome" class="anchor" aria-hidden="true" href="#section-10-expected-outcome"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Section 10 Expected outcome</h2>
<p>To test if the program works, run all the python files. server.py and getData.py will be run on the
AWS EC2 side. The following is the link to the video demonstration of what the application should
look like. <a href="https://www.youtube.com/watch?v=UOrKWCmZnu8" rel="nofollow">https://www.youtube.com/watch?v=UOrKWCmZnu8</a></p>
<p><strong>Entry Gates</strong> should be able to populate the database entry, retrieve available lots value, and show
recommended lots. It should only populate the table or show the lot if the car is not in the carpark.</p>
<p><strong>Parking Lots</strong> should be able to detect if the car is there if the RFID card is detected and the light
level is low. Once this occurs, the light should turn red and the Parking lot table will have a current
holder (value of UID), the available lots for the carpark should also decrease. Once the car leaves,
the reverse will happen where the available lots will go back up and the parking lot will appear as
None after the 20 seconds timeout.</p>
<p><strong>Exit Gates</strong> should change the database entry‚Äôs status of the car from active to inactive. It should
also be able to display the price and allow free entry if the car is not in the carpark for long. This
price will then be inserted into the database entry.</p>
<p><strong>The Website</strong> should be able to be accessed from any device and allow users to retrieve their
current lot id and view their parking history by inputting in the value of the rfid card (value of uid).</p>
